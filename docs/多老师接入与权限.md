# 多老师接入与权限（本地 CLI 每人一套）

> 面向：把 teacher-mcp 开发给其他老师使用时的**账号 / 登录 / 权限**落地指南。
> 部署形态（用户 2026-07-08 拍板）= **本地 CLI 每人一套**：每个老师在自己机器上跑 Claude Code，各自 `.mcp.json` 填自己账号。
> 🔴 核心区分：**工具视图（ROLE）≠ 数据权限（账号角色）**，见 §3。

## 1. 登录不是内嵌的

`login(username, password)` 工具**运行期传真账号**——`.env` 里的账号只是本地单人兜底，不是写死身份。多老师 = 每人用自己的账号。两种传法（本地 CLI 都可）：

- **省事版**：`.mcp.json` 的 env 填 `RUOYI_USERNAME/RUOYI_PASSWORD`，agent 无参调 `login()` 自动用兜底。密码明文在本地配置里——仅限老师自己的机器。
- **安全版**：env 不填密码，每次会话让 agent `login(username, password)` 手动传（或 agent 问老师）。凭据不落配置文件。

## 2. 老师账号从哪来 = 平台普通用户账号（自助注册）

老师账号**就是他在网站登录的普通用户账号**，与 teacher-mcp 解耦——MCP 不建账号，只是拿这个账号 `login`。产生路径：

- **主路 = 自助注册**：老师在网站首页注册弹窗（PRD-C-212 后登录/注册均为页内弹窗）填用户名/密码/昵称+验证码 → `POST /teacher/user/register`（RegisterTeacherBo）。🔴 **系统自动绑 `teacher` 角色**（BE `TEACHER_ROLE_KEY="teacher"` 写死），data_scope=5 数据自动隔离，无需人工赋角色。
- 次路（运维批量建）：RuoYi 后台用户管理页 / `POST /system/user`，绑 `role_key=teacher`。
- 改密：无 FE 入口走 `PUT /system/user/profile/updatePwd`；内部测试账号约定初始 `123456`。
- 🔴 **admin（superadmin）只留给数据管理/运维**（官方讲义库 uid1 owner、跨老师维护）——**绝不发给普通老师**。
- 接入 MCP：老师注册好后，把该账号填进自己机器的 `.mcp.json`（§4），或运行期 `login` 传。

## 3. 🔴 ROLE（工具视图）≠ 账号角色（数据权限）

两个正交维度，别混：

| 维度 | 由什么定 | 控制什么 | 例 |
|---|---|---|---|
| **工具视图** | `.mcp.json` 的 `TEACHER_MCP_ROLE` | 这个连接**能看到哪些工具** | `data`=录入工具组 / `all`=全工具 |
| **数据权限** | **账号的系统角色**（teacher/superadmin） | 能操作**谁的数据** | teacher=仅本人 / superadmin=全部 |

- `ROLE=all` **不是超管**——只是工具全可见；数据仍受账号 data_scope 限制。
- 超管 = 账号是 `superadmin`，与 ROLE 无关。
- 结论：老师账号一律 `teacher`（数据自动隔离），ROLE 按他的**职责**给（见 §4）。

## 4. 每台机器：.mcp.json 模板

按老师职责选 ROLE（工具面收窄=防误操作，不是安全边界，见 §5）：

| 老师职责 | TEACHER_MCP_ROLE | 能用的工具 |
|---|---|---|
| 录题/录卷/录讲义 | `data` | 录入全套 + 共享 |
| 备课 | `prep` | 备课链 + 共享 |
| 举一反三 | `variant` | 变式全链 + 共享 |
| 身兼数职 | `all` | 全部（仍非超管） |

```jsonc
{
  "mcpServers": {
    "teacher-mcp": {
      "command": "d:\\...\\codeplace-O\\teacher-mcp\\.venv\\Scripts\\python.exe",
      "args": ["-m", "teacher_mcp.server"],
      "env": {
        "TEACHER_MCP_ROLE": "data",          // 按职责，见上表
        "RUOYI_USERNAME": "<该老师账号>",     // 省事版填；安全版留空、运行期 login
        "RUOYI_PASSWORD": "<该老师密码>"
      }
    }
  }
}
```

## 5. 当前权限边界（诚实口径，务必知情）

| 隔离层 | 现状 | 说明 |
|---|---|---|
| **数据范围**（看谁的数据） | ✅ **真生效** | teacher 账号 data_scope=5，service 按 create_by，老师只见自己录的 |
| **工具视图**（看哪些工具） | ✅ 有 | MCP ROLE 收窄，防误操作 |
| **功能权限**（能不能调某端点） | ❌ **未拦** | `/teacher/**` 零 `@SaCheckPermission`，技术上任一登录账号可调所有端点 |

🔴 **红线**：ROLE 视图是"防误操作的方便层"，**不是安全边界**——换 ROLE 或裸调 HTTP 可穿透。当前"本地 CLI 每人一套 + 只发 teacher 账号 + 视图收窄"在**内部小范围够用**；但**一旦要开放给外部机构 / 上 web 托管**，必须先做「BE 接口级权限收窄」卡（给 `/teacher/**` 加权限注解 + 权限码 seed，改 C 线 book-server，独立立项独立验证）。此卡当前状态=**暂不做**（用户 2026-07-08 拍板）。
