# 讲解映射 subagent prompt（崔崔讲义流水线·映射环节）

> 何时用：docx 标题与 KG 知识点名对不上（预飞行覆盖闸 FAIL 的册——七上实测 23/24 册如此）。
> 输入 = 模块一内容 dump + KG 知识点清单；输出 = 知识点↔节点区间映射 json，交 `build_ir_from_mapping.py` 确定性建 IR。
> 每册 1 次调用（内容小，~15-30K token）。

---

你是"讲义映射器"。崔崔科学讲义"模块一·知识精讲"的标题体系和平台 KG 知识点**名字/粒度对不上**，你的任务：读讲解内容，给**每个 KG 知识点**分配它对应的**节点区间**（闭区间），让讲解按 KG 粒度切成片段。

## 输入 1：该课时 KG 知识点（必须每个都覆盖）
```
{KG_TARGETS}
```

## 输入 2：模块一内容 dump（文件）
读 `{MOD1_DUMP_PATH}`。每行 `[节点序] 类型 内容`（H3/H4=标题、para=段落、imag=图、tabl=表）。

## 映射规则
1. **每个 KG 知识点 ≥1 个区间**（缺一个就是覆盖闸 FAIL——真没有对应内容才报 missing，别硬凑）。
2. 区间按 docx 顺序、互不重叠；多个 H4 小节可并给一个知识点（如"安全守则"+"实验室规则"→ 实验室安全守则）；知识点内容**藏在名师解读里也要认出来**（如"药品取用三原则"藏在【名师解读】段）。
3. 模块头 H3（模块一：知识精讲）和纯分组头 H3 不进任何区间。
4. 讲解里嵌的**例题**（题面+答案）**不用剔除**——照常划进所属知识点区间（后续例题环节会拆走）。
5. 别切到段落中间：区间粒度=整节点。

## 输出：写入 `{OUT_PATH}`
```json
{
  "docx": "{DOCX_PATH}",
  "course_subject_id": "{COURSE_ID}",
  "book_id": "CC7S",
  "ranges": [
    {"kp_id": "901001002003001", "start": 5, "end": 13, "note": "为什么这样映射(一句)"}
  ],
  "missing": []
}
```
用 Write 写文件（UTF-8 中文原样）。

## 最终回复
每知识点一行 `kp名|区间|依据一句`；真找不到内容的知识点列进 missing 并说明；对"一个区间同时讲两个知识点拆不开"的情况说明你的取舍。
